# cc-statusline — идейный док

Анти-Kickbacks: вместо рекламы в спиннере/статус-баре Claude Code выводим что-то
реально полезное. Живой документ — дополняем по ходу.

---

## 0. Технические рамки (определяют, что вообще можно выводить)

**`statusLine`** — команда, которую Claude Code запускает **на каждом рендере**.
- На stdin приходит JSON сессии, stdout (первая строка) = строка статуса. Поддерживает ANSI-цвета.
- ⚠️ Должна укладываться в **~30–50 мс**. Всё дорогое (сеть, CI, npm audit) — **только из кэша**.
- Дорогие данные кладёт в кэш-файл фоновый обновлятор или хук (`Stop` / `PostToolUse` / `SessionStart`), статус-лайн просто читает готовое.

**`spinnerVerbs`** — **статический** массив строк в `settings.json`, читается при старте сессии.
- Спиннер циклит по массиву, но сам массив за сессию не меняется.
- «Живая» ротация (TODO, что меняешь на лету) → массив должен **перегенерировать хук на `SessionStart`** из файла-источника. Это не per-render, а per-session.

**Что Claude Code даёт на stdin бесплатно:**
`model.{id,display_name}` · `workspace.{current_dir,project_dir}` · `cwd` ·
`cost.{total_cost_usd,total_duration_ms,total_api_duration_ms,total_lines_added,total_lines_removed}` ·
`exceeds_200k_tokens` · `transcript_path` (JSONL сессии — оттуда токены/сообщения/cache-хиты) ·
`version` · `session_id` · `output_style.name`.

**Легенда стоимости:** 🟢 instant (из stdin/память) · 🟡 cheap (локальный shell/fs <30мс) · 🔴 cached (фоновое обновление + чтение кэша)

## 1. Контекст и интеллект сессии

- ⭐ **MVP — Остаток контекста.** 🟢 Парсим `transcript_path` → токены → `▓▓▓▓░░ 68% · 64k left`. Видно, когда пора `/compact`.
- **ETA до авто-compact.** 🟢 По темпу роста: «~8 ходов до лимита».
- **Скорость роста контекста.** 🟢 Токенов/сообщение — замечать, когда сессия «разбухает».
- **Топ-пожиратель контекста.** 🟢 Какой tool-output съел больше всего токенов: «biggest hog: 1×Read = 40k».
- **Cache hit ratio.** 🟢 Из usage-полей транскрипта (`cache_read` vs `input`) — работает ли prompt caching: «cache 92%».
- **Счётчик сообщений/ходов.** 🟢 `msg 47 · turn 18`.
- **Идл-детектор.** 🟢 «Время с последнего твоего сообщения» — заметить зависшую/брошенную сессию.
- **Был ли фолбэк модели.** 🟢 Менялась ли модель за сессию (не сполз ли Opus на меньшую).

## 2. Стоимость / бюджет

- **Стоимость + burn rate.** 🟢 `$0.42 · ~$2.10/ч`.
- **Прогноз до конца.** 🟢 Экстраполяция трат при текущем темпе.
- **Дневной/недельный лимит.** 🔴 Агрегат по всем сессиям из `~/.claude` логов: `$4.20 / $10 today` с предупреждением.
- **Цена за принятую строку.** 🟢 `cost / lines_added` — метрика эффективности.
- **Сравнение со средней сессией.** 🔴 «дороже обычного ×1.8» — аномалия по тратам.
- **Разбивка cache vs fresh токены.** 🟢 Сколько денег экономит кэш.

## 3. Git / VCS

- **Ветка + грязь.** 🟡 `⎇ main ±3`.
- **Ahead/behind upstream.** 🟡 `↑2 ↓0`.
- ⭐ **Возраст последнего коммита.** 🟡 `last commit 3h ago` — пинок «закоммить уже».
- **Непушнутые коммиты.** 🟡 `unpushed: 3`.
- **Untracked-файлы.** 🟡 Сколько новых файлов Claude создал, а ты не добавил.
- **Staged vs unstaged.** 🟡 Раздельный счётчик.
- **Divergence от main.** 🟡 «23 behind main — rebase?».
- **Merge/rebase в процессе.** 🟡 Индикатор конфликта/незавершённого rebase.
- **Detached HEAD warning.** 🟡 Не потеряй работу.
- **Stash count.** 🟡 Напоминание про забытые заначки.
- **Строки в diff, что Claude написал за сессию.** 🟢 vs общий diff — сколько «своего».

## 4. Качество кода / здоровье проекта

- ⭐ **Статус typecheck/тестов.** 🔴 Хук пишет результат `tsc`/теста в кэш → `✓ tsc` / `✗ 3 errors`.
- **Build status + дельта бандла.** 🔴 `build ✓ · +12kb`.
- **Lint errors.** 🔴 Счётчик из кэша.
- **Покрытие тестами + дельта.** 🔴 `cov 78% ▲2`.
- **Новые TODO/FIXME за сессию.** 🟡 Diff-based — что ты насорил по ходу.
- **npm audit / уязвимости.** 🔴 `2 high` из кэша.
- **Устаревшие зависимости.** 🔴 `5 outdated`.
- **Пропавшие env-ключи.** 🟡 Сравнить `.env` vs `.env.example` → «missing: STRIPE_KEY».

## 5. Время / фокус / самочувствие

- ⭐ **Таймер сессии + перерыв.** 🟢 `95 min · встань, разомнись`.
- **Pomodoro с циклами.** 🟢 `🍅 3 · 12:40 left`.
- **20-20-20 для глаз / осанка / вода.** 🟢 Мягкие напоминалки по таймеру.
- **«Кодишь с HH:MM».** 🟢 + сколько суммарно за сегодня.
- **Поздний час.** 🟢 `01:30 — пора спать?`.
- **Тайм-бокс задачи.** 🟢 Ты ставишь «30 мин на это» → обратный отсчёт.
- **Прогресс дневной цели.** 🟡 `задач 3/5`.
- **Стрик коммитов.** 🔴 `🔥 7 дней`.

## 9. Идеи для `spinnerVerbs` (ротация вместо рекламы)

> Помни ограничение из §0: массив статичный, читается при старте. Динамику даёт
> только перегенерация хуком на `SessionStart`.

**Статичный набор (без хука, просто хороший массив):**
- Подзабытые хоткеи/фичи Claude Code (`/compact`, `esc esc` для отката, `#` для памяти, `!` для shell…).
- Чек-лист «перед коммитом»: «прогнал тесты?», «убрал console.log?», «обновил README?».
- Анти-прокрастинация / короткие мантры фокуса.
- Напоминания о здоровье: вода, осанка, глаза.

**Динамичный набор (хук на `SessionStart` генерит массив из файла):**
- ⭐ Твои **TODO из файла** по очереди вместо рекламы.
- Дневная цель/намерение, которое ты задал утром.
- Пинок по git-состоянию: «последний коммит 3ч назад», «3 непушнутых коммита».
- Чек-лист текущего проекта (берётся из `.cc-statusline/spinner.txt` в репо).
- Флеш-карточки (язык/факты) — учишь, пока крутится спиннер (spaced repetition).
- «Дней до дедлайна X: 4».

## 10. Архитектура (решено: модули + конфиг)

```
cc-statusline/
  src/
    statusline.mjs      # энтрипоинт: читает stdin, рендерит включённые виджеты
    config.mjs          # какие виджеты вкл/выкл, порядок, разделитель, цвета
    cache.mjs           # чтение/запись кэша для 🔴-виджетов (TTL)
    widgets/
      context.mjs       # ⭐ MVP
      git.mjs
      cost.mjs
      checks.mjs        # tsc/тесты из кэша
      focus.mjs         # таймер/перерывы
      ...
  hooks/
    refresh-ci.mjs      # фоновое обновление 🔴-кэша (CI, audit…)
    gen-spinner.mjs     # SessionStart → перегенерация spinnerVerbs из TODO-файла
  test/
    sample-input.json
  config.json           # пользовательская настройка: список и порядок виджетов
```

**Контракт виджета:** `async (data, ctx) => string | null` — получает stdin-JSON и
контекст (cwd, cache), возвращает готовый раскрашенный кусок или `null` (скрыт).
Энтрипоинт собирает непустые через разделитель. Каждый виджет изолирован,
падение одного не роняет строку (try/catch вокруг каждого).

## 11½. 🐣 claudegochi (режим тамагочи — реализован в MVP)

Питомец, чьё настроение = состояние сессии. Включается одним полем:
`{ "mode": "tamagotchi", "petName": "claudegochi" }`.

**Сейчас работает:**
- **3-строчный ASCII-спрайт кота** (statusLine рендерит каждую строку отдельным рядом — подтверждено офиц. докой). Компактный однострочный режим — `"petStyle": "compact"`.
- **Сытость = заполненность контекста.** Контекст забивается → питомец голодает → морда `( x_x ) ПОКОРМИ! /compact`.
- **Сердечки + цветная шкала сытости** = остаток «жизни» (свободный контекст): `♥♥♥♥♥ ████████░░` → `♡♡♡♡♡ ░░░░░░░░░░`.
- **Ночной режим** по локальному времени: `( -_- ) поздно, спатки? zZ`.
- Цвет тела по порогу (зелёный → жёлтый → красный), сердечки красные, имя жирным.
- Морды по настроению: `^_^` сыт · `o_o` ок · `;_;` голодный · `x_x` голодает · `-_-` спит.

**Куда растить:**
- **Эволюция/стадии** по «прожитому» (кол-во коммитов за сессию, строк): яйцо → птенец → взрослый.
- **Реакции на события:** закоммитил → `(•‿•)✓ молодец!`; тесты упали → `(╥﹏╥)`; запушил → счастлив.
- **Голод от трат**, а не только контекста: дорогая сессия → «дорого меня кормить».
- **Усталость = длительность сессии**: 3 часа без перерыва → питомец вымотан, просит перерыв.
- **Аксессуары/настроение от стрика** коммитов (🔥 N дней).
- **Кормление = реальное действие:** после `/compact` сердечки восстанавливаются — видимая награда.
- **Мини-анимация** через `spinnerVerbs`: пока крутится спиннер, питомец «шевелится» (ротация кадров кадров морды).
- Несколько питомцев/скинов, выбор через `petName`/`petSkin`.
